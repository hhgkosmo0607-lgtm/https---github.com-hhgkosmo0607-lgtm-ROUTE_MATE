# RouteMate 프로젝트 요약

RouteMate는 여행 계획을 쉽게 만들도록 도와주는 웹서비스다. 사용자가 가고 싶은 장소를 고르면, 일정 순서를 정하고, 이동 시간을 계산하고, 문제가 생겼을 때 대체 일정까지 다시 짜준다.

## 한 줄 설명

여행 장소를 고르고, 하루 일정으로 묶고, 가장 덜 돌아가는 순서로 정리해 주는 여행 도우미 서비스다.

## 핵심 기능

- 일정 자동 정리: 여러 장소를 하루별로 나누고, 이동이 적은 순서로 정리한다.
- 지도에서 장소 찾기: 지도를 보면서 장소를 검색하고 일정에 바로 넣을 수 있다.
- 이동 시간 계산: 자동차나 도보로 얼마나 걸리는지 계산한다. 정확한 계산이 안 되면 대략적인 값으로 대신한다.
- 일정 수정: 장소 순서를 바꾸거나 다른 날로 옮기고, 체류 시간을 직접 바꿀 수 있다.
- 맞춤 추천: 사용자의 취향과 조건을 보고 어울리는 장소를 추천한다.
- Plan B: 가게가 쉬는 날, 너무 오래 기다려야 하는 날, 비 오는 날처럼 계획이 깨질 때 대체 일정을 만든다.
- 부가 기능: 체크리스트, 예산 관리, 공유 링크, 동행자 초대, 여행 복사 기능이 있다.
- 보안: 로그인 보호, 비밀번호 재설정, 요청 횟수 제한, 안전한 정보 저장 같은 기본 보호 장치를 넣었다.

## 지도 API 비교

| 선택지 | 장점 | 단점 | 이 프로젝트에 맞는 점 |
|---|---|---|---|
| 네이버 지도 | 한국 장소 검색이 익숙하고 국내 정보가 강하다 | 보통 API 키와 사용 조건이 필요하다 | 국내 사용자에게 친숙한 화면을 만들기 좋다 |
| 구글 지도 | 전 세계 지도와 길 찾기 품질이 좋다 | 비용과 정책 제한을 확인해야 한다 | 해외 장소까지 넓게 다루기 좋다 |
| OSM 계열(Leaflet + OpenStreetMap + Nominatim + OSRM) | 무료로 시작하기 쉽고 구조가 단순하다 | 지역에 따라 장소 정보가 부족할 수 있다 | 현재 프로젝트 구조와 가장 잘 맞는다 |

정리하면, 길 찾기 기능은 지도 화면만 있으면 되는 게 아니라 장소 검색과 경로 계산까지 같이 필요하다. 그래서 네이버나 구글 같은 상용 지도 API를 쓸 수도 있고, 지금처럼 OSM 계열 조합으로도 만들 수 있다.

## 전체 그림

```text
				   사용자 입력
			    여행할 장소를 고름
					│
					▼
			화면 코드와 화면 기술
	   [코드] app/controllers/*, app/templates/*, app/static/js/*
	   [기술] Flask, Jinja2, JavaScript, CSS
					│
					▼
			   장소 찾기와 지도 표시
      [코드] place_controller.py, place_service.py,
	      adapters/map_adapter.py, adapters/osm_adapter.py
      [기술] Leaflet, OpenStreetMap, Nominatim, Overpass
					│
					▼
			   일정 만들기와 순서 정리
      [코드] trip_controller.py, schedule_controller.py,
	      route_engine.py, route_optimizer.py
      [기술] Python, SQLAlchemy, Google OR-Tools, OSRM
					│
					▼
			   추천과 대체 일정 처리
      [코드] recommendation_controller.py, recommend_service.py,
	      recommender.py, planb_engine.py
      [기술] 규칙 기반 추천, Plan B 재구성, Haversine 폴백
					│
					▼
			   저장과 보안 처리
      [코드] models/*.py, utils/crypto.py, utils/audit.py, app/__init__.py
      [기술] SQLite / MySQL, bcrypt, AES-256-GCM, CSRF, CSP, Rate Limit
					│
					▼
				  결과 화면 출력
			  완성된 여행 일정과 상태 표시
```

이 그림은 사용자가 장소를 고르는 순간부터, 화면 표시, 지도 검색, 일정 만들기, 추천, Plan B, 저장과 보안까지 이어지는 흐름을 보여준다. 각 단계마다 실제 코드 파일과 사용 기술을 함께 적었다.

## 기술 구성

- Python: 이 프로젝트의 기본 프로그래밍 언어로, 전체 기능을 만드는 데 사용한다.
- Flask: 웹사이트와 API를 만드는 가벼운 웹 프레임워크다.
- SQLAlchemy: 데이터베이스를 코드로 다루게 해주는 도구다.
- SQLite: 개발할 때 쓰는 간단한 데이터 저장소다.
- MySQL: 실제 서비스에서 사용할 수 있는 더 큰 데이터 저장소다.
- Jinja2: 서버에서 화면에 보여줄 HTML을 만들어 주는 템플릿 도구다.
- JavaScript: 브라우저에서 버튼 클릭, 지도 조작 같은 움직임을 처리한다.
- CSS: 화면의 색, 배치, 글자 모양을 꾸미는 역할을 한다.
- OpenStreetMap: 지도 화면에 쓰는 공개 지도 데이터다.
- Leaflet: 지도를 웹페이지에 띄우고 조작하게 해주는 지도 라이브러리다.
- Nominatim: 장소 이름을 검색해서 지도 좌표로 바꿔주는 기능이다.
- Overpass: 지도 주변의 장소 정보를 찾아오는 도구다.
- OSRM: 두 장소 사이의 실제 도로 이동 거리와 시간을 계산하는 도구다.
- Google OR-Tools: 여러 장소의 순서를 더 효율적으로 정해 주는 최적화 도구다.
- Docker: 실행 환경을 한 번에 묶어서 어디서나 비슷하게 실행하게 해준다.
- Gunicorn: Flask 앱을 실제 서버처럼 돌려 주는 실행 도구다.
- GitHub Actions: 코드 점검과 테스트를 자동으로 돌려 주는 도구다.

## 주요 폴더

- `app/engines/`: 일정 정리, 추천, Plan B처럼 핵심 계산을 하는 부분
- `app/adapters/`: 지도 서비스와 외부 정보를 연결하는 부분
- `app/services/`: 실제 서비스 흐름과 작업 처리를 담당하는 부분
- `app/controllers/`: 화면 요청과 API 요청을 받는 부분
- `app/models/`: 여행, 일정, 장소, 사용자 정보를 저장하는 부분
- `app/templates/`, `app/static/`: 화면 파일과 디자인 파일
- `tests/`: 기능이 잘 동작하는지 확인하는 테스트
- `loadtest/`: 사용자가 많을 때 얼마나 버티는지 보는 부하 테스트

## 실행과 테스트

- 실행: 필요한 라이브러리를 설치한 뒤 데이터베이스를 준비하고 서버를 실행하면 된다.
- 테스트: `pytest`로 검사하고, 더 자세한 확인은 E2E 테스트와 정적 검사로 진행한다.

## 현재 상태 요약

- 기본 기능은 대부분 만들어져 있다.
- 일부 외부 서비스는 아직 완전히 연결되지 않았고, 그 경우를 대비한 대체 방식이 들어 있다.
- 로컬에서 외부 서비스 없이도 실행되고 테스트할 수 있게 준비돼 있다.
