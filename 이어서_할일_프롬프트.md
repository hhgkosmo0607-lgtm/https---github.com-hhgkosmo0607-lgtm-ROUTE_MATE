### 남은 일 ①  ← 여기부터 이어서
**"Day 분배 기준이 이상하다"** — 경로 자동 생성 시 날짜별 장소 분배가 부자연스럽다
(먼 곳끼리 같은 날에 묶이거나, 하루에 너무 적게/많게 들어감).

내가 세운 가설(아직 검증 전):
`app/engines/route_optimizer.py`의 VRPTW 모델에서 **depot(하루 시작/끝) 연결 비용이 0**이라
(`transit_cb`에서 `if i == depot or j == depot: return 0`), 날짜를 쪼개면 이동 arc가 공짜로
하나 사라진다 → 솔버가 총 이동시간을 줄이려고 **불필요하게 날을 잘게 나누는** 유인이 생긴다.
또한 Day 간 부하를 균형 잡는 항이 목적함수에 전혀 없다.

제안하는 수정 방향:
1. `routing.SetFixedCostOfAllVehicles(...)` — 날(=차량)을 하나 더 쓰는 데 비용을 매겨 공짜 분할 차단
2. `time_dim.SetGlobalSpanCostCoefficient(...)` — 가장 긴 Day를 페널티로 눌러 Day 간 부하 균형

**먼저 할 일: 고치기 전에 현재 분배 결과를 측정해서 뭐가 실제로 잘못됐는지 확인해줘.**
아래 스크립트로 재현 가능:
```bash
.venv/Scripts/python -c "
from datetime import time
from app.engines import route_engine as re
P=re.PlaceInput
places=[P(place_id=1,lat=37.5796,lng=126.9770,stay_min=90),
        P(place_id=2,lat=37.5759,lng=126.9769,stay_min=60),
        P(place_id=3,lat=37.5735,lng=126.9850,stay_min=60),
        P(place_id=4,lat=37.5636,lng=126.9834,stay_min=90),
        P(place_id=5,lat=37.5512,lng=126.9882,stay_min=90),
        P(place_id=6,lat=37.5445,lng=127.0557,stay_min=60),
        P(place_id=7,lat=37.5563,lng=126.9236,stay_min=60),
        P(place_id=8,lat=37.5219,lng=126.9245,stay_min=60)]
r=re.build_route(places,'CAR',3,time(9,0),time(21,0))
for d in r.days: print('Day',d.day_no,[i.place_id for i in d.items])
print('미배치',r.unassigned,'총이동',r.total_move_min,'분')
"
```
측정 결과를 보고 원인을 확정한 뒤 수정하고, `tests/test_route_optimizer.py`에 회귀 테스트를 추가해줘.
